# 修改计划：解决学生赋值覆盖测试数据的问题

**日期**：2026-06-07

## 问题

Scratch 学生程序在绿旗后手动赋值输入变量（如 `设 n 为 5`），会覆盖判题器注入的测试数据。

## 方案：替换输入变量的首次赋值字面量

### 原理

对于每个输入变量，只处理其在执行顺序中的**第一次赋值**：
- VALUE 是字面量 → 替换为测试数据
- VALUE 是表达式 → 跳过，保留原样
- 后续对该变量的所有赋值 → 不处理，保留原样

### 示例

```
学生代码：设 n 为 5, 设 ans 为 n * n, 设 n 为 0
测试数据：n=10

处理过程：
  设 n 为 5     → n 的第一次赋值，VALUE 是字面量 → 替换为 10
  设 ans 为 n*n → 不是输入变量的赋值 → 跳过
  设 n 为 0     → n 的第二次赋值 → 跳过（保留）

运行：n=10, ans=10*10=100, n 被设为 0
```

### 实现要点

1. 遍历所有 target 的 blocks，找到脚本头部（topLevel=true 或 SUBSTACK 入口）
2. 沿 `next` 指针按执行顺序遍历积木链
3. 遇到 `data_setvariableto` 且目标是输入变量：
   - 如果该变量是第一次遇到 → 标记已处理，替换 VALUE 中的字面量
   - 如果已处理过 → 跳过
4. VALUE 判断：压缩格式中 `[1, [4, val]]` 或 `[1, [10, val]]` 是字面量

### 验证

测试程序：绿旗 → 设 n 为 5 → 设 ans 为 n * n → 设 n 为 0
测试用例 {n: 10} → 期望 ans = 100（替换首次赋值后 n=10）
